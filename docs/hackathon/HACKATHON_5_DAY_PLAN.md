# 5-Day Delivery Plan

## Day 1

- freeze the MVP shape
- confirm the paid artifact catalog
- implement payment-intent and verification interfaces
- implement the first Arc adapter surface
- define the exact live demo path

## Day 2

- bridge paid jobs into Nexus orchestration
- create explicit lifecycle states:
  - requested
  - awaiting_payment
  - paid
  - queued
  - in_progress
  - in_review
  - delivered
- persist audit events

## Day 3

- build the buyer-facing UI
- expose pricing, payment state, job state, and delivery unlock
- wire at least one real artifact type from ToTheMoonTokens

## Day 4

- harden the vertical demo
- add test coverage and demo evidence
- remove broken flows
- improve the visual hierarchy of the demo screen

## Day 5

- run the full demo repeatedly
- finalize screenshots
- finalize architecture summary
- finalize short and long submission copy
- record or script the demo video

## Non-negotiable rules

- no live trading
- no mainnet execution
- no huge refactor detours
- no broad marketplace ambitions beyond the MVP
- one end-to-end story beats many disconnected partial features

## Success condition

By the deadline, the team must be able to show:

1. a priced agent action
2. a validated payment state
3. a Nexus job in motion
4. a review gate
5. an unlocked final artifact
6. a clear explanation of why Arc + Circle matter economically
